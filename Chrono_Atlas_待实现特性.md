## 特性 1：历史密度地图（时间线鸟瞰）

- **方向**：数据探索
- **规模**：M
- **评审总分**：4.75 / 5.0
- **描述**：在时间线顶部添加一条全局"事件密度地图"——把 6000 年压缩成一条窄带，用颜色深浅表示每个时段的事件密度，用户可以一眼看到哪些时期"热闹"、哪些时期"空白"。
- **实现思路**：新增 `TimelineDensityMap.tsx` 组件。把时间轴分成 120 个等宽桶（每桶 50 年），计算每桶事件数，用 CSS 渐变渲染。点击某个桶跳转到对应年份。在 TimelineView 顶部接入。使用当前筛选的事件数据，实时响应。
- **风险点**：极端筛选下密度太稀疏；需要最小显示粒度
- **评审备注**：这是最高价值的特性——给用户一个 6000 年的"上帝视角"。建议桶宽度自适应事件数量范围。点击桶时调用 `setFocusYear`。组件应该很轻量（纯 CSS 渐变 + 事件监听），不影响性能。

## 特性 2："如果你在那里"沉浸式视角卡片

- **方向**：视觉叙事
- **规模**：M
- **评审总分**：4.55 / 5.0
- **描述**：在 EventDetail 中新增一段第二人称叙事——"如果你在公元前 450 年的雅典，你会看到……"，用沉浸式文字把用户拉进历史现场。
- **实现思路**：在 `event-detail.ts` 新增 `generateImmersiveNarrative()` 函数。根据事件的 category、region、year 和 significance 组合生成第二人称叙事模板。在 EventDetail 中用一个独特的渐变卡片渲染。
- **风险点**：模板化太明显会显得尴尬
- **评审备注**：关键在于模板多样性。建议按 (时代段 × 类目) 构造矩阵，至少准备 30+ 个变体模板。使用事件 ID hash 做确定性选择（和现有 "你知道吗" 一致的模式）。渲染位置建议在 "你知道吗" 区块之前。

## 特性 3：事件连连看（记忆配对游戏）

- **方向**：交互体验
- **规模**：M
- **评审总分**：4.45 / 5.0
- **描述**：一个历史事件配对小游戏——翻牌匹配"事件名↔年份"，训练历史记忆力。
- **实现思路**：新增 `MemoryMatch.tsx` 组件。从 filteredEvents 中随机选 6 个事件生成 12 张牌，CSS 3D 翻转动画。记录翻牌次数和用时，结束后评级。
- **风险点**：事件数太少时需要 fallback
- **评审备注**：选事件时优先选里程碑事件（更有辨识度）。评级建议 3 档：★（完成）、★★（<20 次翻牌）、★★★（<16 次翻牌）。App.tsx 中添加入口按钮，lazy load。

## 特性 4：历史事件情绪色谱条

- **方向**：视觉叙事
- **规模**：S → M（组件相对简单）
- **评审总分**：4.35 / 5.0
- **描述**：为每个时代在 TimelineView 中渲染一条"情绪色谱"条——根据该时代事件的类目分布，用类目色彩比例条可视化该时代的"基调"。
- **实现思路**：新增内联组件或独立 `EraSpectrumBar` 在 TimelineView 的时代分隔区域。按当前时代事件的 category 统计占比，渲染为一条 CSS 渐变比例条。hover 显示 tooltip。
- **风险点**：事件数太少的时代色谱条意义有限
- **评审备注**：建议设置最小阈值（≥5 个事件才渲染色谱条）。色谱条高度 4px，不影响整体布局。颜色直接用 CATEGORY_CONFIG.color。

## 特性 5：因果链动画播放器

- **方向**：视觉叙事
- **规模**：M
- **评审总分**：4.35 / 5.0
- **描述**：把因果链从静态展示变成可播放的"故事线"——点击播放后，因果链上的事件逐个高亮，配合叙事文案自动滚动。
- **实现思路**：在 `CausalNetworkGraph.tsx` 中增加播放状态。用 setInterval 逐个激活节点，CSS 动画控制高亮效果。底部同步展示 `buildCausalNarrative()` 文案。
- **风险点**：因果链长度差异大（3-22 节点），播放时间需自适应
- **评审备注**：播放间隔建议：短链（≤5）3 秒/步，中链（6-12）2 秒/步，长链（>12）1.5 秒/步。提供暂停和逐步前进按钮。激活节点用脉冲动画（CSS @keyframes pulse）。

## 特性 6：时代环境音景

- **方向**：视觉叙事
- **规模**：S
- **评审总分**：3.80 / 5.0
- **描述**：在事件详情页根据时代自动显示一个"环境标签"——用文字描述该时代的感官氛围。
- **实现思路**：在 `event-detail.ts` 新增 `getEraAmbience()` 函数，为 10 个时代各准备 2-3 条氛围描述（按事件 category 或 region 细分），用 hash 做确定性选择。在 EventDetail 底部以半透明卡片渲染。
- **风险点**：氛围描述措辞需要平衡学术性和趣味性
- **评审备注**：建议氛围描述保持简短（一行），使用感官词汇（声音、气味、触感），避免夸张。渲染位置在延伸阅读和推荐之间。

## 特性 7："历史冷知识弹幕"模式

- **方向**：交互体验
- **规模**：S
- **评审总分**：3.80 / 5.0
- **描述**：在时间线视图添加一个"弹幕"开关——开启后，随机冷知识以弹幕形式从右向左飘过时间线。
- **实现思路**：新增 `DanmakuOverlay.tsx` 组件。从 fun-facts 和 generateDidYouKnow 提取冷知识池，每 3-5 秒发射一条弹幕。CSS animation 控制飘动。弹幕可点击跳转。
- **风险点**：弹幕可能遮挡时间线内容
- **评审备注**：弹幕默认关闭，需要用户主动开启。同屏最多 3 条。透明度 0.6，hover 时 1.0。弹幕区域限制在时间线上方 1/3 区域。localStorage 保存开关状态。
