# Chrono Atlas 架构约束规则

> 此文件由 product-evolve Skill 读取。Agent 在实现任何特性前必须遵守以下规则。
> 违反规则的变更必须被拒绝并修正。

## 目录结构约束

| 文件类型 | 必须放在 | 禁止放在 |
|---------|---------|---------|
| React 页面/视图组件 | `app/src/components/` | 项目根目录、server/ |
| React hooks | `app/src/hooks/` | components/ 内部 |
| 数据文件（事件/王朝） | `app/src/data/` | components/、hooks/ |
| 类型定义 | `app/src/data/types.ts` | 散落在各组件中 |
| UI 基础组件 | `app/src/components/ui/` | 与业务组件同级 |
| 后端路由/API | `server/src/index.ts` | app/ 下 |
| 数据库操作 | `server/src/db.ts` | index.ts 中内联 |
| E2E 测试 | `e2e/` | app/src/ 内 |
| 静态资源 | `app/public/` | app/src/ |

## 不可修改的核心文件

以下文件的**结构和接口**不可随意修改（可以追加新功能，但不能破坏现有接口）：

- `app/src/data/types.ts` — HistoricalEvent 类型定义是全局契约
- `server/src/db.ts` — 数据库表结构不可随意改
- `server/src/seed.ts` — 种子数据逻辑不可删除

## 组件规范

1. **新视图组件**必须：
   - 接受 `events` 和 `onSelectEvent` props
   - 在 `App.tsx` 的 viewMode 条件渲染中注册
   - 在 `FilterPanel.tsx` 的视图模式列表中添加入口

2. **新 hooks** 必须：
   - 以 `use` 开头命名
   - 导出类型定义

3. **单文件行数上限**：300 行。超过必须拆分。

## 代码风格

- 函数式组件 + hooks（不用 class 组件）
- Tailwind CSS 做样式（不用 CSS Modules / styled-components）
- Lucide React 做图标（不用 emoji 做 UI 图标）
- TypeScript 严格模式
- 所有 props 必须有类型定义

## 禁止的操作

- ❌ 不允许修改 `package.json` 的 `name` 或 `type` 字段
- ❌ 不允许添加新的 CSS 框架（已用 Tailwind，不要引入 Bootstrap/Ant Design 等）
- ❌ 不允许引入 ORM（已用 better-sqlite3 直接查询）
- ❌ 不允许在前端直接操作数据库
- ❌ 不允许删除已有的视图组件
