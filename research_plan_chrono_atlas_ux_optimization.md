# Chrono Atlas 交互优化研究计划

## 目标
基于 `Chrono_Atlas_交互设计优化建议.md`，评估当前实现与建议之间的差距，提炼优先级，并在代码中落地首批高价值优化。

## 问题拆解
本次任务既是研究型问题，也是实施型问题，核心包含四部分：
1. 将 12 条优化建议映射到当前代码实现，判断已实现、部分实现、未实现。
2. 识别最适合本轮直接落地的 P0/P1 优化点，优先选择低风险高收益项。
3. 参考外部设计/交互资料，确认高信息密度时间线产品的合理交互模式。
4. 在前端代码中完成首批优化并验证可用性。

## 查询类型判断
这是一个**Breadth-first query（广度优先）**：
- 优化建议天然拆分为多个独立子问题：首次引导、时间线导航、详情层级、筛选反馈、统计增强、移动端优化等。
- 不同子问题可并行研究，再汇总成统一实施方案。
- 最终答案需要同时包含评估、优先级和代码改造结果。

## 研究方法备选
### 方案 A：仅基于本地文档和代码盘点
优点：快；缺点：缺少外部 UX 依据，容易只做“按文施工”。

### 方案 B：本地代码盘点 + Web 搜索设计资料
优点：能给出更稳妥的交互依据；缺点：中文高质量资料分散。

### 方案 C：本地代码盘点 + Web 搜索 + 微信公众号文章检索
优点：同时利用代码事实、公开 Web 资料和中文高质量公众号文章；更适合中文产品/设计语境。

**采用方案 C。**

## 信息检索计划
### 1. 代码盘点（主线）
- 目标：明确建议点与现状差距，锁定关键文件。
- 关键词：欢迎引导、首次访问、筛选摘要、回到顶部、详情抽屉、模态框、搜索联想、移动端导航。
- 输出：建议点 -> 现状 -> 涉及文件 -> 优先级。

### 2. Web 检索（补充 UX 依据）
- 时间范围：2023-2026。
- 关键词：
  - 高信息密度 时间线 产品 交互 优化
  - 首次引导 欢迎态 UX 最佳实践
  - 长列表 虚拟滚动 时间线 导航
  - Drawer 替代 Modal 详情阅读
  - 筛选状态反馈 可视化 设计
- 输出：适用于本项目的实现原则与注意事项。

### 3. 微信公众号文章检索（必须纳入）
- 技能：`wechat-article-search`
- 时间参数：必须带时间范围。
- 时间范围：`2023-01-01` 到 `2026-03-19`
- 关键词：
  - 时间线 交互设计
  - 首次引导 欢迎页 产品设计
  - 长列表 虚拟滚动 体验优化
  - 抽屉 详情页 交互设计
  - 筛选器 交互反馈 设计
- 使用方式：优先检索中文高质量公众号文章，提炼适合本项目的设计模式；再与 Web 搜索结果交叉验证。

## 子任务分配
### 子任务 1：代码盘点
- 工具：`code-explorer` 子代理
- 输出：12 条建议的实现现状、文件映射、优先级。
- 必要性：高。

### 子任务 2：外部 UX 模式研究
- 工具：`research_subagent`
- 要求：结合 Web 搜索与微信文章思路，聚焦首次引导、欢迎态、长列表导航、抽屉详情、筛选反馈、统计增强。
- 输出：6-10 条可执行结论。
- 必要性：中高。

### 子任务 3：主代理实施规划
- 综合本地现状与外部结论，挑选首批直接落地项。
- 预期选择：欢迎态/轻引导、筛选摘要条、时间线回顶/强化导航、详情路径收敛。

## 首批实施准则
优先选择满足以下条件的项：
1. 对体验改善明显。
2. 主要影响前端组件层，不要求大规模改后端。
3. 不大幅破坏现有交互路径。
4. 能在本轮内完成并验证。

## 验证计划
- 对修改文件执行 lints。
- 运行前端 build。
- 如有必要启动预览检查页面结构。

## 预期交付
1. 优化点评估摘要。
2. 已落地优化说明。
3. 后续建议（如虚拟滚动、统计增强、搜索联想等中期项）。
